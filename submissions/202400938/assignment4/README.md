# Assignment 4: 데이터 수집 및 탐색적 분석 (EDA) 보고서

## 1. 프로젝트 개요

- **프로젝트명**: Aero-Post Generator (항공 뉴스 캡션 자동 생성기)
- **목표**: `AeroRoutes`와 같은 전문 항공 뉴스 사이트의 정형화된 텍스트를 분석하여, 핵심 정보(항공사, 노선, 일정 등)를 추출하는 **NER(개체명 인식) 모델**을 학습시키기 위한 데이터셋 구축
- **해결하려는 문제**: 전문 용어와 약어로 가득 찬 항공 뉴스를 매번 수동으로 해석하고 SNS용 캡션으로 작성하는 비효율적인 반복 작업을 해결하고자 함

## 2. 데이터 수집 (Data Collection)

본 프로젝트는 **Python 기반의 크롤러**를 활용하여 데이터를 수집했습니다.

- **수집 방법**: ‘requests’ 및 ‘BeautifulSoup’ 라이브러리를 이용한 웹 크롤링
- **데이터 소스**: [AeroRoutes](https://www.aeroroutes.com/)
    - 선정 이유: 전 세계 항공 스케줄 변경 정보를 가장 빠르고 정확하게 전달하는 핵심 소스
- **수집 기간**: 2025년 9월 17일 ~ 2025년 11월 23일
- **데이터 크기**: 총 1,000개 (CSV)
- **파일 위치**: ‘data/aviation_data.csv’
- **데이터 포맷**:
    - `source`: 데이터 출처 (AeroRoutes)
    - `title`: 기사 제목 (예: *Air Arabia Lists Sharjah – London Gatwick...*)
    - `content`: 기사 본문 텍스트

## 3. 데이터 분석 및 주요 발견 (EDA Findings)

### 3.1 기초 통계 및 분포
- **텍스트 길이**: 평균 약 **581자**, 중앙값 **410자**
    - 대부분의 기사가 300~600자 사이에 밀집해 있으며, 이는 팩트 위주의 간결한 문장 구조를 가지고 있음을 의미함
    - 최대 길이 12,000자 이상의 이상치(Outlier, 주간 요약글 등)가 일부 존재하여, 향후 학습 시 전처리가 필요함을 확인하였음

### 3.2 주요 발견 사항
- **구조적 패턴(Structure)**: 데이터의 대부분이 ‘[항공사] + [노선] + [날짜/횟수] + [기종]’ 순서의 매우 규칙적인 패턴을 보임
- **핵심 키워드(Keyword)**: 빈도 분석 결과, ‘weekly’, ‘aircraft’, ‘winter’ 등 스케줄 변경과 관련된 특정 동사와 명사가 압도적으로 많이 등장함
- **데이터 품질**: 무작위 수집임에도 불구하고 ‘Incheon’, ‘Seoul’, ‘Korea’ 등 국내 관련 키워드가 포함된 기사들이 다수 존재하여, 타겟 독자(동아시아/한국)에게 유효한 정보임을 확인함

## 4. 향후 계획 (Assignment 5)

- **데이터 전처리**:
    - 길이이가 너무 긴 이상치 제거
    -  Published at…과 같은 메타 데이터 등의 노이즈 제거
- **모델 학습**:
    - 수집된 1,000개의 정제된 데이터를 활용하여 ‘[AIRLINE]’, ‘[ROUTE]’, ‘[DATE]’, ‘[AIRCRAFT]’ 등을 추출하는 **BERT 기반 Token Classification 모델**을 학습시킬 예정
    - 학습된 모델은 추후 LLM API와 결합할 예정
