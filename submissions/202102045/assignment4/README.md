## 1. 프로젝트 개요

### 1.1 프로젝트 주제
**국제/정치 뉴스 요약 봇 (Global & Politics News Summarizer)**

### 1.2 목표
* 매일 발생하는 방대한 양의 국제 정세 및 정치 관련 뉴스를 수집
* 핵심 내용을 **'3줄 요약'** 형태로 자동 변환하여 사용자에게 제공하는 AI 서비스 개발

### 1.3 필요성
* **정보 과잉 해결:** 사용자가 긴 기사를 모두 읽지 않고도 주요 정치/외교 이슈를 신속하게 파악할 수 있도록 지원

## 2. 데이터 수집 (Data Collection)

본 프로젝트는 초기 웹 크롤링 방식의 불안정성(접속 차단, 구조 변경 등)을 해결하고, 지도 학습(Supervised Learning)을 위한 **고품질 정답 데이터(Label)** 확보를 위해 오픈소스 데이터셋 활용으로 전략을 변경했습니다.

* **수집 방법:** Hugging Face `datasets` 라이브러리를 활용한 API 로드

    * *특징: 네이버 뉴스 기사 원문과 사람이 검수한 요약문 쌍으로 구성*

## 3. 데이터 분석 및 주요 발견 (EDA)

수집된 **22,194건**의 데이터에 대한 기초 통계 및 시각화 분석 결과입니다.

### 3.1 기초 통계
* **데이터 크기:** 총 22,194건 (Training Set 기준)
* **텍스트 길이 분포:**

| 구분 | 평균 길이 | 분석 결과 |
| :--- | :--- | :--- |
| **본문** | 약 998자 | 대부분 500~1,500자 분포 |
| **요약문** | 약 183자 | BERT/BART 입력 허용량 내에서 **정보 손실 없이 학습 가능** |

### 3.2 ⚠️ 주요 발견 사항 및 문제점

#### 1) 심각한 도메인 편향 (Domain Bias)
* **현상:** '정치', '국회', '대통령' 키워드가 제목에 포함된 기사는 **단 133건 (전체 0.6%)**에 불과함.
* **문제점:** 데이터가 경제, IT, 생활문화에 편중되어 있어 국제 정세나 정치적 맥락(외교 프로토콜, 법안 발의 등) 학습에 **데이터가 턱없이 부족함.**

#### 2) 데이터 품질 및 이상치 (Quality & Outliers)
* **현상 (역전 현상):** 요약문의 길이가 본문보다 긴 논리적 오류 데이터 **1,739건 (약 7.8%)** 발견.
* **문제점:** 전처리 없이 학습 시, 모델이 요약 대신 내용을 창작하거나 원문을 복사하는 **환각(Hallucination) 현상** 유발 위험.

## 4. 향후 계획 (Future Plan: Assignment 5)

EDA를 통해 파악된 **'정치 데이터 부재'**와 **'이상치'** 문제를 해결하기 위한 모델링 전략입니다.

### 4.1 데이터 확보 전략 
1.  **대체 데이터셋 도입**
2.  **실시간 파이프라인 구축:**
    * 추론(Inference) 단계에서 네이버 뉴스 API를 활용해 실시간 '정치/세계' 뉴스를 수집 및 주입하는 자동화 파이프라인 구축.

### 4.2 모델 학습 전략
1.  **전처리 (Preprocessing):**
    * 이상치 제거: 요약문이 본문보다 긴 데이터(1,739건) 및 50자 미만 초단문 데이터 전량 **Drop**.
2.  **전이 학습 (Transfer Learning):**
    * **Stage 1 (General Ability):** 기존 데이터(2.2만 건)로 일반적인 '한국어 문장 압축 능력' 학습.
    * **Stage 2 (Domain Adaptation):** BBC 기반 정치 뉴스(`xlsum`)로 정치/외교 도메인에 특화된 **미세 조정(Fine-tuning)** 수행.
